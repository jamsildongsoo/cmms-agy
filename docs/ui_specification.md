# UI 구성 방안 (UI Specification)

본 문서는 CMMS의 프론트엔드 라우팅 맵, 화면 레이아웃 정의, 그리고 핵심 컴포넌트의 화면 와이어프레임(Mockup)을 기술합니다. 
본 명세는 실무 집중도를 높인 **차분하고 정갈한 비즈니스 룩**을 지향하며, **태블릿 환경**과 **인쇄 레이아웃**에 최적화되어 있습니다.

---

## 1. 프론트엔드 라우팅 및 메뉴 구조

*   **대시보드**: 최초 요구사항 검토 단계에서 **구현 보류**로 결정되었습니다. (추후 필요 시 추가)
*   **라우팅 맵**:
    ```text
    / (로그인 화면)
    └── /app (메인 레이아웃 - Side Navigation 적용, 태블릿 대응 축소형 제공)
        ├── /mdm (기준정보 관리)
        │   ├── /company (회사 정보 - SYSTEM 권한 전용)
        │   ├── /plant (플랜트 정보 - Read-Only for manager)
        │   ├── /department (부서 정보 - 조직도 트리, Read-Only for manager)
        │   ├── /user (사용자/권한 관리, Read-Only for manager)
        │   ├── /warehouse (창고 관리)
        │   └── /code (공통코드 그룹 및 아이템 관리)
        ├── /master (마스터정보 관리)
        │   ├── /equipment (설비 마스터 CRUD, 점검주기/항목 설정, 가로 스크롤/열고정, 다운로드)
        │   └── /inventory (재고 마스터 CRUD, 리스트 다운로드)
        ├── /transaction (업무 트랜잭션)
        │   ├── /pm (예방점검 기록 입력 및 결재 상신/전결)
        │   ├── /wo (작업지시 계획/실적 등록 및 결재 상신/전결)
        │   ├── /wp (작업허가서 작성, 체크시트 JSON 입력 및 결재 상신/전결)
        │   └── /stock (부품 입출고/이동 처리 및 재고 현황/입출고장 출력)
        ├── /approval (전자결재함)
        │   ├── /draft (상신함)
        │   ├── /inbox (결재/합의함)
        │   └── /reference (참조함)
        ├── /profile (마이페이지 - 개인정보 조회 및 비밀번호 수정)
        └── /board (게시판 - 공지사항 상단 고정, 댓글)
    ```

## 2. 레이아웃 및 디자인 정책 (Tablet & Professional Clean Look)

### 2.1. 태블릿(Tablet) 최적화 반응형 디자인
*   **사이드바 자동 축소**: 가로 해상도 `768px` 이상 `1024px` 이하의 태블릿 환경에서는 좌측 사이드바가 **아이콘 형태의 미니 바(Mini-Sidebar)**로 자동 축소되어, 본문 그리드와 입력 폼의 넓은 화면 영역을 확보합니다.
*   **터치 타겟 확장**: 태블릿 터치 실수를 방지하기 위해 모든 버튼과 입력 폼 요소의 세로 높이를 충분히 확보(`h-10` ~ `h-12` 내외)하고, 패딩 값을 넉넉히 제공합니다.

### 2.2. 전문 업무용 테마 (Slate-Gray Look)
*   화려한 디자인 요소를 최대한 절제하고 업무에만 몰입할 수 있는 **차분한 톤앤매너**를 유지합니다.
*   **색상 체계**: 백그라운드는 순백색(`white`) 또는 밝은 회색(`slate-50`), 경계선은 얇은 회색(`slate-200`), 폰트는 짙은 회색(`slate-800`). 포인트 컬러는 집중도를 높이는 네이비/블루 계열(`slate-900`, `blue-700`)로 제한.
*   **글꼴**: Pretendard 또는 나눔고딕 계열의 고대비(High-contrast) 한글 서체를 사용하여 가독성을 최우선으로 합니다.

### 2.3. 인쇄/출력(Print View) 정책
*   화면상의 배경색은 인쇄 시 모두 배제하고 **순백색 배경**에 짙은 검은색 텍스트와 얇은 경계선만으로 깨끗하게 출력되도록 `@media print` 스타일을 강제 적용합니다.
*   인쇄 시 불필요한 스크롤바, 사이드바, 헤더, 액션 버튼은 강제 숨김(`display: none`) 처리합니다.

## 3. 핵심 화면 와이어프레임 (Wireframes)

### 3.1. 통일화된 거래(Transaction) 문서 헤더 양식 (예방점검, 작업지시, 작업허가)
*   **예방점검**, **작업지시**, **작업허가** 화면 및 출력본은 **우측 상단의 4x2 결재 박스를 표시하지 않고**, 문서 기본 정보와 **[문서상태]**, **[연계결재번호]** 정보만 깔끔한 테이블 형태로 통일되게 보여줍니다.

### 3.3. 가로형 그리드 및 열 고정 (Frozen Column)
*   설비 목록, 재고 목록, 이력 화면 등에서 컬럼이 많아 가로 스크롤이 발생할 때, 식별 코드열은 좌측에 항상 고정(Sticky)됩니다.

```text
[가로 스크롤 영역]
+-----------------------------------------------------------------------------------+
| [고정열] | (가로 스크롤 가능 영역 ◀─────────────────────────────────────────────▶) |
+----------+------------------------------------------------------------------------+
| 설비코드 | 설비명      | 설비타입 | 설치일자   | 가동여부 | 제조사   | 일련번호   | ... |
+----------+-------------+----------+------------+----------+----------+------------+-----+
| EQ-0001  | 보일러 B-101| 압력용기 | 2020-01-01 | 가동중   | 한국중공 | SN-103948  | ... |
| EQ-0002  | 공급 펌프 P1| 펌프     | 2021-06-15 | 정지     | 현대정밀 | SN-902384  | ... |
+----------+-------------+----------+------------+----------+----------+------------+-----+
```

### 3.4. 작업허가서(Work Permit) 아코디언 통합 스크롤 구조
*   **배경 및 문제점**: 작업허가서는 여러 종류(화기, 밀폐 등)를 중복 선택할 수 있습니다. 탭(Tab) 구조로 설계하면 화면 입력은 편리할 수 있으나, 브라우저에서 인쇄할 때 보이지 않는 탭 영역의 데이터가 출력에서 누락되는 심각한 문제가 발생합니다.
*   **해결 제안 (아코디언 카드 스크롤 방식)**:
    *   **일반작업(기본)**은 항상 상단에 열려 있는 카드로 고정 노출됩니다.
    *   나머지 특수 작업(화기, 밀폐 등)은 체크박스 선택 시 아래에 **개별 카드(세로 아코디언 형태)로 순차적으로 나타납니다.**
    *   각 카드는 제목 우측에 접기/펴기 버튼을 가집니다.
    *   **인쇄 시(Print Mode)**: 모든 아코디언 카드를 **강제로 강제 전개(Expand All)**하고, 각 작업 종류별로 **페이지 나누기(Page Break)**를 자동 실행하여 일반 표지 뒤로 화기작업서, 밀폐작업서 등이 개별 종이 문서로 한 장씩 깔끔하게 인쇄되도록 구현합니다.

```text
+-----------------------------------------------------------------------------------+
|  작업허가서 작성 (아코디언 구조)                                                  |
|                                                                                   |
|  * 허가유형: [v] 일반(필수)  [v] 화기작업  [ ] 밀폐공간  [v] 정전작업  [ ] 고소작업  |
|                                                                                   |
|  ===============================================================================  |
|  [카드 1] 일반 안전조치 요구사항 (상시 노출)                                       |
|  - [v] 안전모 및 안전화 착용 완료                                                 |
|  - [v] 경고 표지판 설치 완료                                                      |
|  ===============================================================================  |
|  [카드 2] 화기작업 안전조치 요구사항 (화기 선택 시 활성화)                           |
|  - [ ] 작업구역 인화물질 제거 완료                                                |
|  - [ ] 소화기 비치 완료                                                           |
|  ===============================================================================  |
|  [카드 3] 정전작업 안전조치 요구사항 (정전 선택 시 활성화)                           |
|  - [ ] 주 차단 스위치 내림 및 잠금 확인                                            |
|  - [ ] 방전 및 전압 잔류 여부 검전 확인                                            |
+-----------------------------------------------------------------------------------+
```

## 4. 인쇄/출력 및 반응형 정책

1.  **가로형 그리드 인쇄 최적화**:
    *   설비 목록, 재고 목록, 점검 상세 등의 가로형 출력 양식은 CSS `@media print` 규칙을 통해 브라우저 인쇄 창 호출 시 네비게이션 Sidebar 및 각종 기능 버튼이 자동으로 숨겨지고 가로 인쇄(Landscape) 모드로 레이아웃이 자동 조절됩니다.
2.  **공통 헤더 인쇄 정보**:
    *   인쇄 문서 상단에 아래 양식을 동적으로 렌더링합니다:
    `[회사명: (주)한국설비]    |    [출력자: 홍길동 과장]    |    [출력일시: 2026-05-25 10:15]`

## 5. 권한 등급별 UI 노출 및 기능 제어 정책

사용자 권한 등급(`role_id`)에 따라 프론트엔드 네비게이션 및 기능 버튼의 노출 상태를 다음과 같이 제어합니다.

### 5.1. `system` (시스템 총괄 관리자)
*   **네비게이션**: 오직 '회사 등록 관리' 및 '시스템 로그/이력' 화면만 노출되며, 일반 CMMS 메뉴(설비, 재고, 트랜잭션 등)는 숨김 처리됩니다.
*   **기능 제어**: 신규 회사(Company) 생성 버튼이 유일하게 활성화됩니다.

### 5.2. `admin` (회사 최고 관리자)
*   **네비게이션**: 소속 회사 내의 모든 메뉴(조직 정보, 마스터 정보, 트랜잭션, 결재, 게시판)가 노출됩니다.
*   **기능 제어**: 조직 정보 및 기준정보 관리 화면의 모든 등록/수정/삭제 버튼이 활성화됩니다.

### 5.3. `manager` (부서 관리자)
*   **네비게이션**: `admin`과 동일하게 모든 메뉴가 노출됩니다.
*   **기능 제어**:
    *   **조직 정보 관리**: 조회만 가능하도록 제한되며, 모든 저장/수정/삭제 버튼이 비활성화(Disabled)되고 "조회 전용 권한입니다." 안내 문구가 표시됩니다.
    *   **업무 트랜잭션 (예방점검, 작업지시, 작업허가)**: **[직접확정저장 (S)]** 버튼이 노출 및 활성화되어 결재 상신 없이 즉시 확정 처리가 가능합니다.

### 5.4. `user` (일반 사용자)
*   **네비게이션**: 기준정보 관리(조직도 제외)는 숨김 처리되거나 조회만 가능하며, 마스터 및 트랜잭션 메뉴 위주로 노출됩니다.
*   **기능 제어**: 
    *   **업무 트랜잭션**: **[직접확정저장 (S)]** 버튼이 숨김 처리되며, 반드시 [결재상신 (P)] 또는 [임시저장 (T)]만 수행할 수 있습니다.
