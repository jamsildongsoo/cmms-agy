# CMMS 개발 계획

소규모 사업장용 CMMS(설비관리시스템) 개발 구현 계획서(초안이며, 실제 문서는 별도 작성함)

본 프로젝트는 30인 미만의 소규모 사업장을 대상으로 하는 웹 기반 설비관리시스템(CMMS) 구축을 목표로 합니다. 

> [!NOTE]
> 본 문서는 AI 시스템의 UI 렌더링용 메타데이터 폴더(`~/.gemini/antigravity-cli/brain/...`)와 함께, 프로젝트 소스 제어 및 관리를 위해 프로젝트 루트(`projects/cmms-agy/docs/implementation_plan.md`)에도 복사본이 유지됩니다.

## 구현 기능

### 0. 공통 
- 헤더는 soft delete이고 아이템은 hard 
- 출력은 헤더에 회사명,출력자이름,출력열자(YYYYMMDD:HHmm) 공통표시

### 1. 로그인 및 인증
- 아이디, 패스워드로 로그인하며, access token 부여하고 화면에서 expire 기간 연장 가능
- FE에서 useStore로 [회사,플랜트,아이디,이름,권한,직책] 관리

### 2. 조직정보 
- 회사,플랜트,부서,권한,사용자,저장소,코드(그룹-아이템) 등록(수정), 리스트조회, 항목조회
- 부서는 계층도로 구조화

### 3. 마스터정보
- 설비 마스터: 등록(수정), 리스트조회, 항목조회, 리스트출력(가로), 리스트다운로드
- 재고 마스터: 등록(수정), 리스트조회, 항목조회, 리스트출력(가로), 리스트다운로드

### 4. 트랜잭션정보 
- 예방점검: 설비마스터점검주기에서 다음점검일 기준으로 목록에서 조회하고 점검결과 입력하며, 기준날짜는 오늘이지만 사용자가 선택 가능. 대상(반려,임시저장포함) 목록 조회하며, 결재연계-임시저장, 결재상신(결재선 선택+완결시C), 권한자는 직접 확정(S) 기능 제공함. 항목별 상세 출력 및 목록 출력(가로)
- 작업지시 : 계획을 수립하고 결재(C) 또는 저장(S)에 대해 결과 입력, 결재연계-임시저장, 결재상신(결재선 선택+완결시C), 권한자는 직접 확정(S) 기능 제공함.기본 프로세스는 예방점검과 동일.항목별 상세 출력 및 목록 출력(가로)
- 작업허가 : 계획을 수립하며 결재연계-임시저장, 결재상신(결재선 선택+완결시C), 권한자는 직접 확정(S) 기능 제공함. 복수유형 선택가능하고 일반은 항상 선택. 각 항목별 별도의 양식이 존재함. 출력은 일반표지외 각 항목별로 페이지 분리하여 출력함.항목별 상세 출력 및 목록 출력(가로)
- 재고처리 : 재고아이디별로 입고,출고 처리하고 입고 시 평균법으로 stock관리. 이동은 저장소 간 처리하며 출고-입고 동시 처리하고 이력테이블은 개별 행으로 처리하고 입고이력에 출고번호를 참조번호로 입력. 재고입출고는 건별 또는 목록으로 입력. 건별 재고 입고/출고장 출력하고 목록에서 가로 현황 출력기능. 결재연계 없음

### 5. 게시판
- 일반 게시판이며, 공지여부 선택하면 리스트에서 최상단 고정. 댓글 가능하고 1단계(다단계 아님)

### 6. 결재
- 일반결재처리이며, 결재자+유형(결재,합의,참조)선택 후 상신
- 결재 출력 양식 제공하고 오른쪽 상단에 결재박스 4칸씩 2열 제공하고 1열은 결재자,2열은 합의자. 2열 하단에 참조자 리스트 표기. 1/2열 각 박스는 상단에 직책, 그 밑에 이름, 그 밑에 결재일시(YYYYMMDDmm)
- 결재함은 상신함,결재함(합의포함),참조함으로 구성
- 결재항목은 항목번호 0은 상신자, 1부터 결재순번으로

## 기술 스택 상세

### 1. Backend (Spring Boot)
*   **Java Version**: Java 21 (LTS)
*   **Build Tool**: Gradle (Kotlin DSL)
*   **Framework**: Spring Boot 3.3.x
*   **Security & Auth**: Spring Security + JWT (JSON Web Token)
*   **Database Access**: Spring Data JPA + Querydsl
*   **Migration**: Flyway (데이터베이스 형상 관리)

### 2. Frontend (React)
*   **Build Tool**: Vite
*   **Language**: TypeScript
*   **State Management**: Zustand
*   **Styling**: Tailwind CSS (버전은 사용자 피드백에 따라 v3 또는 v4 적용 예정)
*   **HTTP Client**: Axios

### 3. Infra (Docker)
*   **Database**: PostgreSQL 16
*   **Web Server / Reverse Proxy**: Nginx
